# Домашнее задание

**Тема:** Начало создания прототипа игры

Так как наш курс про геймдизайн, то логично предположить, что итогом нашей работы будет прототип игры. В оставшихся домашних работах мы будем подходить к этой цели. Тематика прототипа выбирается по желанию (пример: шутер, рогалик, RPG и т.д.).

## Цели задания

- Создать репозиторий проекта в [GitLab](https://gitlab.com/gitlab-com).
- Создать проект Blueprint на базе движка Unreal Engine 5.2 и добавить его в репозиторий.
- Создать ветку для текущего ДЗ.
- Добавить описание проекта. Нужно написать, о чём будет ваша игра.

## Для зачёта предыдущих ДЗ

- Создать уровень главного меню.
- Создать виджет главного меню (должны быть кнопки: «Выйти из игры», «Начать игру», «О игре»).
- Создать GameMode главного меню.
- Сделать переход с главного меню на основной уровень по нажатию кнопки «Начать игру».
- По нажатию кнопки «Выйти из игры» игра должна закрываться, либо останавливаться симуляция.
- По нажатию кнопки «О игре» должен выводиться текст с приветствием игрока и описанием игры.

## Создать основной уровень игры

- Сделать отдельный lvl.
- Добавить несколько Static Mesh (можно использовать любые ассеты, но лучше не использовать слишком «тяжёлые» 4K-модели с текстурами, заточенными под DirectX).
- Сделать уровень многоярусным. Можно использовать любые примитивы, которые есть в движке. Не нужно наводить красоту и добавлять много ассетов.
- Добавить несколько `PlayerStart`. Они должны быть расположены на разных ярусах и быть привязаны к логике прохождения уровня.

## Создать главного героя

- Можно использовать Pawn, Character.
- Настроить герою управление.
- Можно импортировать любые модели, главное — чтобы они подходили под стандартный скелет.
- Добавить персонажу несколько `ActorComponent` (прим. HealthComponent).

### Анимации для персонажа

- Импортировать любой набор бесплатных анимаций.
- Создать AnimBP.
- Добавить его для главного героя.

## Создать PickableActor

- Персонаж должен уметь взаимодействовать с предметами и объектами на сцене.
- Должны быть поднимаемые предметы (прим. оружие, аптечка, collectables).
- Должны быть квестовые объекты; взаимодействие с ними нужно для прохождения уровня (прим. дверь, ключ и т.д.).

## Сдача работы

- Сделать коммит, запушить изменения в репозиторий.
- Подготовить MR для сдачи работы:
  - В MR выставить флаг «Squash commits».
  - Название MR должно в двух словах описывать суть задачи.
  - В описании MR нужно описать, что было сделано — это нужно для упрощения проверки.
- Ссылку на MR приложить в LMS для проверки домашнего задания.

> Для получения максимальной оценки нужно также добавить противников. Логику их действий будем описывать позже. Пока нужно подготовить Pawn, AnimBP и Mesh.
