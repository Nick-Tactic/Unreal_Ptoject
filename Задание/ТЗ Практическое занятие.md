# Техническое задание: «Domino Run»

**Цель:** создать мини-сцену с ландшафтом и физикой (шар запускает домино), запуск через главное меню.

---

## 1) Структура проекта

Создать папки в `Content`:

- `Maps`
- `Blueprints`
- `Widgets`

Создать 2 уровня и сохранить в `Content/Maps`:

- `MainMenu`
- `DominoRun`

---

## 2) Главное меню (уровень MainMenu)

### 2.1 Виджет меню

Создать Widget Blueprint в `Content/Widgets`:

- `WBP_MainMenu`

На виджете разместить:

- Заголовок (например, **DOMINO RUN**) — по желанию
- Кнопка **Start**
- Кнопка **Quit**

### 2.2 Логика кнопок

В `WBP_MainMenu`:

| Кнопка | Действие |
|--------|----------|
| **Start** (OnClicked) | Open Level → `DominoRun` |
| **Quit** (OnClicked) | Quit Game |

### 2.3 Подключение меню на уровне

На уровне `MainMenu` (Level Blueprint):

1. Event BeginPlay
2. Create Widget (`WBP_MainMenu`)
3. Add to Viewport
4. Set Input Mode UI Only
5. Show Mouse Cursor = True

### 2.4 Стартовая карта проекта

**Project Settings → Maps & Modes:**

- Editor Startup Map = `MainMenu`
- Game Default Map = `MainMenu`

**Критерий:** при запуске проекта открывается меню, Start ведёт на сцену, Quit закрывает игру.

---

## 3) Игровая сцена (уровень DominoRun)

### 3.1 Ландшафт

На уровне должен быть Landscape (можно базовый/небольшой) с ровной областью под трассу домино.

**Требования к рельефу:**

- есть ровная площадка (под домино);
- есть небольшие перепады высот вокруг (для вида).

---

## 4) Объекты сцены и параметры

### 4.1 Шар-стартер

Создать объект **Sphere**:

| Параметр | Значение |
|----------|----------|
| Static Mesh | Sphere |
| Location | X=220, Y=5500, Z=1570 |
| Rotation | 0, 0, 0 |
| Scale | 3, 3, 3 |
| Mobility | Movable |

**Физика шара:**

| Параметр | Значение |
|----------|----------|
| Simulate Physics | ON |
| Enable Gravity | ON |
| Collision Preset | PhysicsActor |
| Use CCD | ON |
| Linear Damping | 0.0–0.05 |
| Angular Damping | 0.0–0.1 |

**Критерий:** шар падает и запускает цепочку домино.

### 4.2 Домино (кубики)

Создать домино из **Cube** (все одинаковые).

**Общие параметры для всех домино:**

| Параметр | Значение |
|----------|----------|
| Static Mesh | Cube |
| Scale | X=3, Y=1, Z=10 |
| Mobility | Movable |
| Simulate Physics | ON |
| Enable Gravity | ON |
| Collision Preset | PhysicsActor |
| Use CCD | ON (желательно) |
| Z (высота) | подобрать так, чтобы кубики стояли на поверхности (ориентир Z≈500, как в примере) |

---

## 5) Расстановка домино (координатные правила)

### 5.1 Первые домино

| Объект | Координаты |
|--------|------------|
| Domino_01 | X=220, Y=5320, Z=500 |
| Domino_02 | X=220, Y=4820, Z=500 |

### 5.2 Основная линия до развилки

Все домино в линии:

- X = 220
- Z = 500
- шаг по Y = 500 между соседними

**Формула (если нужно):**

```
Y = 5320 - 500 * (i - 1)   для Domino_i
```

---

## 6) Развилка на 2 ветки

После домино в точке развилки построить две ветки.

### 6.1 Правила

- шаг по Y в каждой ветке остаётся **500**;
- ветки расходятся по X:
  - левая ветка: X = 220 − 250 → **X_left = −30**
  - правая ветка: X = 220 + 250 → **X_right = 470**

### 6.2 Шаблон координат веток

Пусть `Y_split` — координата последнего общего домино. Тогда для k-го домино каждой ветки:

**Левая ветка:**

- X = −30
- Y = Y_split − 500 × k
- Z = 500

**Правая ветка:**

- X = 470
- Y = Y_split − 500 × k
- Z = 500

---

## 7) Критерии приёмки

- Меню запускается первым, кнопки работают.
- В `DominoRun` шар падает и запускает домино.
- Домино падают последовательно, без «зависаний».
- Развилка активирует обе ветки.
- Соблюдены расстояния:
  - шаг между домино по линии: **500**
  - смещение веток по X: **±250**
